# Project Summary
Nolimit360 is a design management platform that enhances the process of submitting, tracking, and managing design requests. It offers a streamlined experience for clients and designers, facilitating efficient collaboration and providing real-time updates on project statuses.

# Project Module Description
- **Backend**: Manages database interactions, authentication, and CRUD operations for design requests. It has been updated to handle optional fields correctly in the request schema.
- **Frontend**: Provides a user interface for clients and admins to manage design requests, view statistics, and interact with the design team. Recent updates have improved form handling to avoid sending `null` values for optional fields.

# Directory Tree
```
app/
├── backend/
│   ├── core/                  # Core functionalities including authentication and database management
│   ├── models/                # Database models for requests, users, and brands
│   ├── routers/               # API endpoints for handling requests
│   ├── services/              # Business logic for managing requests
│   ├── dependencies/          # Dependency injection and authentication
│   ├── alembic/               # Database migrations
│   └── main.py                # Entry point for the backend application
├── frontend/
│   ├── src/                   # Source code for the frontend application
│   ├── public/                # Static assets like images and favicon
│   ├── components/            # Reusable UI components
│   └── pages/                 # Individual pages for the application
└── start_app_v2.sh            # Script to start the application
```

# File Description Inventory
- **app/backend/main.py**: Entry point for the backend application.
- **app/backend/routers/**: Contains API routes for managing requests and user authentication, including updated handling for optional fields.
- **app/backend/models/**: Defines the database schema, now correctly typing optional fields.
- **app/frontend/src/**: Contains the frontend application code, including pages and components, with recent improvements to form submission.
- **app/frontend/public/**: Stores public assets such as images and stylesheets.

# Technology Stack
- **Backend**: FastAPI, SQLAlchemy, PostgreSQL
- **Frontend**: React, TypeScript, Tailwind CSS, Vite

# Usage
1. **Install Dependencies**:
   - For the backend: Navigate to `app/backend` and run the dependency manager to install required packages.
   - For the frontend: Navigate to `app/frontend` and run the dependency manager to install required packages.

2. **Build the Application**:
   - For the backend: Use the command to build the backend application.
   - For the frontend: Use the command to build the frontend application.

3. **Run the Application**:
   - Start the backend server.
   - Start the frontend server.
