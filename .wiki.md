# Project Summary
Nolimit360 is an advanced design management platform that streamlines the process of submitting, tracking, and managing design requests. It provides a comprehensive solution for clients and designers, enabling efficient collaboration, real-time updates, and a user-friendly interface for managing design assets and team interactions.

# Project Module Description
- **Backend**: Handles database interactions, authentication, and CRUD operations for design requests. It includes new features such as managing brands, notifications, and design files.
- **Frontend**: Offers a dynamic user interface for clients and admins to manage design requests, view statistics, and collaborate with the design team. Recent enhancements include AI-assisted request submissions and an improved dashboard.

# Directory Tree
```
app/
├── backend/
│   ├── core/                  # Core functionalities including authentication and database management
│   ├── models/                # Database models for requests, users, brands, and notifications
│   ├── routers/               # API endpoints for handling requests, brands, and notifications
│   ├── services/              # Business logic for managing requests and notifications

│   ├── alembic/               # Database migrations
│   ├── data_models/           # JSON files for brand data

├── frontend/
│   ├── src/                   # Source code for the frontend application
│   ├── public/                # Static assets like images and favicon
│   ├── components/            # Reusable UI components
│   ├── pages/                 # Individual pages for the application
│   └── todo.md                # Enhancement plan and tasks

```

# File Description Inventory
- **app/backend/main.py**: Entry point for the backend application.
- **app/backend/routers/**: Contains API routes for managing requests, brands, and notifications.
- **app/backend/models/**: Defines the database schema for requests, users, brands, and notifications.
- **app/frontend/src/**: Contains the frontend application code, including pages and components with recent improvements.
- **app/frontend/public/**: Stores public assets such as images and stylesheets.
- **app/frontend/src/todo.md**: Document outlining the enhancement plan and tasks.

# Technology Stack
- **Backend**: FastAPI, SQLAlchemy, PostgreSQL
- **Frontend**: React, TypeScript, Tailwind CSS, Vite

# Usage
1. **Install Dependencies**:
   - For the backend: Navigate to `app/backend` and run the dependency manager to install required packages.
   - For the frontend: Navigate to `app/frontend` and run the dependency manager to install required packages.

2. **Build the Application**:
   - For the backend: Use the command to build the backend application.
   - For the frontend: Use the command to build the frontend application.

3. **Run the Application**:
   - Start the backend server.
   - Start the frontend server.
