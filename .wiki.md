# Project Summary
Nolimit360 is an advanced design management platform that streamlines the process of submitting, tracking, and managing design requests. It enhances collaboration between clients and designers with real-time updates and a user-friendly interface for managing design assets and team interactions. The platform is designed to improve efficiency and communication in design workflows.

# Project Module Description
- **Backend**: Manages database interactions, authentication, and CRUD operations for design requests. Features include brand management, notifications, and design file handling.
- **Frontend**: Provides a dynamic user interface for clients and admins to manage design requests, view statistics, and collaborate with the design team. Recent updates include the removal of the demo toggle button and the introduction of role-based admin access.

# File Description Inventory
- **app/backend/main.py**: Entry point for the backend application.
- **app/backend/routers/**: Contains API routes for managing requests, brands, and notifications.
- **app/backend/models/**: Defines the database schema for requests, users, brands, and notifications.
- **app/frontend/src/**: Contains the frontend application code, including pages and components with recent improvements.
- **app/frontend/public/**: Stores public assets such as images and stylesheets.
- **app/frontend/src/todo.md**: Document outlining the enhancement plan and tasks.
- **app/frontend/src/contexts/AuthContext.tsx**: Manages authentication context and admin role checks.

# Technology Stack
- **Backend**: FastAPI, SQLAlchemy, PostgreSQL
- **Frontend**: React, TypeScript, Tailwind CSS, Vite

# Usage
1. **Install Dependencies**:
   - For the backend: Navigate to `app/backend` and run the dependency manager to install required packages.
   - For the frontend: Navigate to `app/frontend` and run the dependency manager to install required packages.

2. **Build the Application**:
   - For the backend: Use the command to build the backend application.
   - For the frontend: Use the command to build the frontend application.

3. **Run the Application**:
   - Start the backend server.
   - Start the frontend server.
