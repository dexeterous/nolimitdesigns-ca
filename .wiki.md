# Project Summary
Nolimit360 is an advanced design management platform that simplifies the submission, tracking, and management of design requests. It fosters collaboration between clients and designers through real-time updates and an intuitive interface for managing design assets. The platform aims to enhance efficiency and communication in design workflows, catering to businesses that require streamlined design processes.

# Project Module Description
- **Backend**: Handles database interactions, authentication, and CRUD operations for design requests. Key features include brand management, notifications, and design file handling.
- **Frontend**: Offers a dynamic user interface for clients and admins to manage design requests, view statistics, and collaborate with the design team. Recent updates include enhanced Kanban board functionality, revision tracking, and an improved submit request form.

# File Description Inventory
- **app/backend/main.py**: Entry point for the backend application.
- **app/backend/routers/**: Contains API routes for managing requests, brands, and notifications.
- **app/backend/models/**: Defines the database schema for requests, users, brands, and notifications.
- **app/frontend/src/**: Contains the frontend application code, including pages and components with recent improvements.
- **app/frontend/public/**: Stores public assets such as images and stylesheets.
- **app/frontend/src/todo.md**: Document outlining the enhancement plan and tasks.
- **app/frontend/src/contexts/AuthContext.tsx**: Manages authentication context and admin role checks.

# Technology Stack
- **Backend**: FastAPI, SQLAlchemy, PostgreSQL
- **Frontend**: React, TypeScript, Tailwind CSS, Vite

# Usage
1. **Install Dependencies**:
   - For the backend: Navigate to `app/backend` and run the dependency manager to install required packages.
   - For the frontend: Navigate to `app/frontend` and run the dependency manager to install required packages.

2. **Build the Application**:
   - For the backend: Use the command to build the backend application.
   - For the frontend: Use the command to build the frontend application.

3. **Run the Application**:
   - Start the backend server.
   - Start the frontend server.
